bilibili ijkplyer的编译和使用

发表于 2016-05-24   |   分类于 视频   |     |   阅读次数

前言

ijkplayer是一个基于 ffplay 的轻量级 Android/iOS 视频播放器。实现了跨平台功能，API易于集成；编译配置可裁剪，方便控制安装包大小；支持硬件加速解码，更加省电；提供Android平台下应用弹幕集成的解决方案，此方案目前已用于主流APP，比如美拍和斗鱼直播 。

准备工作

建议在Mac下面编译，本人在Mac下面编译的，用windows去编译会有莫名的错误信息。

系统中确认有以下环境

  1. Git。（在终端里面输入git，有正确的反馈就是说明安装过了，否则可以根据提示自动搜索安装，或者在Git下载安装）
  2. ANDROID_SDK环境并且配置环境变量。（一般andorid studio都下载有这个sdk目录）
  3. ANDROID_NDK环境并且配置环境变量。（得自行下载，官方建议NDK r10e。第一去官方网站下载NDK，第二是我的云盘里面下载了Mac的这个版本，去官网下载需要翻墙，百度云地址，提取码：d7f4）

配置sdk和ndk环境变量

  1. 启动Mac自带的终端Terminal。
  2. 进入当前用户的home目录。
    在命令行输入echo $HOME
  3. 创建.bash_profile文件。
    命令行输入touch .bash_profile
  4. 打开.bash_profile文件。
    命令行输入open -e .bash_profile 此命令行输入完毕后，会自动弹出 .bash_profile 文件的编辑窗口
  5. 在.bash_profile文件中输入 export PATH=/Users/您的用户名/文件路径/platform-tools/:$PATH。注意：如果需要添加多个路径到PATH下时，需要用“:”(英文的冒号)隔开。因为android和adb这些命令是在不同的文件夹中的，格式是：PATH=${PATH}:路径1:路径2、另外，如果指定多个路径，结尾就不用写:$PATH了。例如：export PATH=${PATH}:/Users/yangxixi/Library/Android/sdk/platform-tools:/Users/yangxixi/Library/Android/sdk/tools:/Users/yangxixi/Library/Android/android-ndk-r10e
  6. 保存文件。
    直接关闭.bash_profile文件，即可自动保存您输入的内容。
  7. 更新刚配置的环境变量
    在命令行中输入source .bash_profile
  8. 验证配置是否成功
    输入adb，如果没有出现-bash: adb: command not found；而出现adb的命令 参数解释。则表明环境变量配置成功。

开始编译

按照如下步骤一次进行：

  1. 到克隆之后的主目录，执行 ./init-android.sh
  2. 执行 ./compile-ffmpeg.sh clean
  3. 执行 ./compile-ffmpeg.sh armv7a
  4. 最后执行 ./compile-ijk.sh
  5. 每一个步骤要耐心等待执行完毕，第一个步骤可能下载速度不快，后面编译的步骤等待执行完毕再继续。为什么只编译armv7a平台？是因为大部分的手机平台架构都是这个。
